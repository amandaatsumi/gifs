Code Refactoring

  • Concept
    • Redability: Everyone can easily read and understand your code
    • Modularity: If something stops working, will it be easy to break it down and find a solution
    • Efficiency: How fast the code runs
    • Length: Try to cut the code so it works better for the previous items. Code Golf has some ideas
  • Steps
    1. Keep the same HTML elements together
      • Example h1, h2, h3, h4, h5, h6 {same font-family}. And anything specific is added to a new class
      • Give the class names refering to the section of the website to make it easier to debug
      • Add classes while building the html so you don't have to go back and refactoring later
      • We can combine title and class as #title .class { } and this creates a specific configuration
    2. Combining Selectors types:
      1. Multiple selectors: h1, h2, h3, h4, h5, h6 { }
      2. Differetn selectors: h1, p { }
      3. Hierarchical Selectors: #title .class-name { }
      4. Hirarchical Parent-child syntax: div .title { } - It is read from right to the left
      5. Hirarchical Parent-child syntax: div h1{ } - It is for a h1 inside the div
      6. Hirarchical Parent-child syntax: .container-fluid h1 { } - only h1s inside a class container-fluid element
      7. Combined Selectors no space between:div(class="container-fluid") - they need to be in the same element
      8. With space it reads "container .title {color: red;}" as the title inside the class container should have red color
      9. Without space it reads "container.title {color: red;}" look for a element that has both class and make it red. They need to be inside the same html element

  • Background-color - bg-light/dark/danger/warming

  • Troggler - is used to collapse
  • The structure is NAV > UL > LI
  • To add the links to other parts of the page, it needs to use a ID and anchor tag

Grid

  • Div class="row"
  • Each block is a Div class=col
  • Div class- col-6 takes 6/12 of the line
  • Every line is divided in 12
  • New Div Class=rows are new lines
  • New Div Class=Col are new blocks
  • There are 6 breakpoints: xs, sm, md, lg, xl, xxl, that specify the sizes per device
    • div class Col-lg-3 col-md-4 l-sm-6 means:
      • When in a large device it takes 3/12 columns
      • When in a medium device it takes 4/12 columns
      • When in a small device it takes 6/12 columns
  • When using Google Fonts, you need to import sizes 100, 400 and 900

Carousel

  • First you need to set the layout you want to apply in the carousel section
  • It required and ID to work
  • The structure is:
    1. Div: with ID, a class, "data-ride" informing if the slides move alone (false= it doesn't)

        • Div with Class=carousel-item and at least one of them need to be class=carousel-item active

        • Copy the structure from Bootstrap website

Z-Index

  • Children sit on top of the parent
  • This crops the images
  • All elements have z-index: 0. If we change it to -1 or 1 it changes the order of present
  • Z-index only works for Position: ABSOLUTE

Stacking order

  • The images are displayed per order as it is written in the html code: the last one goes on the top
  • Position and Z-index can change the order

Media Queries

  • @media
  • Queries: check if something is true or false
  • Example 1: @media(max-width: 10px) {h1 { color: red} }: It will make the letter red to smaller screen
  • Example 1: @media(min-width: 10px) {h1 { color: red} }: It will make the letter red to bigger screen

Code Refactoring

  • Concept
    • Redability: Everyone can easily read and understand your code
    • Modularity: If something stops working, will it be easy to break it down and find a solution
    • Efficiency: How fast the code runs
    • Length: Try to cut the code so it works better for the previous items. Code Golf has some ideas
  • Steps
    1. Keep the same HTML elements together
      • Example h1, h2, h3, h4, h5, h6 {same font-family}. And anything specific is added to a new class
      • Give the class names refering to the section of the website to make it easier to debug
      • Add classes while building the html so you don't have to go back and refactoring later
      • We can combine title and class as #title .class { } and this creates a specific configuration
    2. Combining Selectors types:
      1. Multiple selectors: h1, h2, h3, h4, h5, h6 { }
      2. Differetn selectors: h1, p { }
      3. Hierarchical Selectors: #title .class-name { }
      4. Hirarchical Parent-child syntax: div .title { } - It is read from right to the left
      5. Hirarchical Parent-child syntax: div h1{ } - It is for a h1 inside the div
      6. Hirarchical Parent-child syntax: .container-fluid h1 { } - only h1s inside a class container-fluid element
      7. Combined Selectors no space between:div(class="container-fluid") - they need to be in the same element
      8. With space it reads "container .title {color: red;}" as the title inside the class container should have red color
      9. Without space it reads "container.title {color: red;}" look for a element that has both class and make it red. They need to be inside the same html element
